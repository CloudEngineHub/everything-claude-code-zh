# Google Workspace 运维

此技能用于将共享文档、电子表格和演示文稿作为工作系统来操作，而非仅孤立编辑单个文件。

## 何时使用

- 用户需要查找文档、表格或演示文稿并就地更新
- 整合存储在 Google Drive 中的计划、跟踪器、笔记或客户列表
- 清理或重构共享电子表格
- 导入、修复或重新格式化 Google Slides 演示文稿
- 从 Docs、Sheets 或 Slides 生成摘要供决策参考

## 首选工具接口

使用 Google Drive 作为入口，然后切换到正确的专家工具：

- Google Docs 用于文本密集的文档
- Google Sheets 用于表格工作、公式和图表
- Google Slides 用于演示文稿、导入、模板迁移和清理

不要仅从文件名猜测结构。先检查。

## 工作流

### 1. 查找资产

从 Drive 搜索接口开始定位：

- 确切文件
- 同级资产
- 可能的重复项
- 最近修改的版本

如果多个文档看起来相似，按标题、所有者、修改时间或文件夹确认。

### 2. 编辑前检查

在修改之前：

- 摘要当前结构
- 识别标签页、标题或幻灯片数量
- 检测任务是局部清理还是结构性手术

选择能安全完成工作的最小工具。

### 3. 精确编辑

- Docs：使用索引感知编辑，而非模糊重写
- Sheets：在明确的标签页和范围上操作
- Slides：区分内容编辑、视觉清理或模板迁移

如果请求的工作是视觉或布局敏感的，通过检查和验证迭代而非一次性大规模盲更新。

### 4. 保持工作系统整洁

当文件是更大工作流的一部分时，还应指出：

- 重复的跟踪器
- 过时的演示文稿
- 过期文档 vs 规范文档
- 资产是否应归档、合并或重命名

## 输出格式

使用：

```text
资产
- 文件名
- 类型
- 为什么这是正确的文件

当前状态
- 结构摘要
- 关键问题或阻碍

行动
- 已做或建议的编辑

后续步骤
- 归档 / 合并 / 重复清理 / 下一个需要更新的文件
```

## 适用场景

- "找到活动的规划文档并精简它"
- "清理这个客户电子表格，显示流失风险行"
- "将这个演示文稿导入 Slides 并使其可演示"
- "找到当前的跟踪器，不是过期的重复项"
