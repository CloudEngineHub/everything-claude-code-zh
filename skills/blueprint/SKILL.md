# Blueprint — 构建计划生成器

将单行目标转变为任何编码智能体可以冷启动执行的分步构建计划。

## 何时使用

- 将大型功能分解为具有清晰依赖顺序的多个 PR
- 规划跨多个会话的重构或迁移
- 协调子智能体之间的并行工作流
- 任何会话之间的上下文丢失会导致返工的任务

**不要用于**可在单个 PR、少于 3 个工具调用中完成的任务，或当用户说"就去做"时。

## 工作原理

Blueprint 运行 5 阶段管道：

1. **研究** — 预检查（git、gh 身份验证、远程、默认分支），然后读取项目结构、现有计划和内存文件以收集上下文。
2. **设计** — 将目标分解为一个 PR 大小的步骤（通常 3–12 个）。分配依赖边、并行/串行排序、模型层（最强与默认）和每步的回滚策略。
3. **草拟** — 将独立的 Markdown 计划文件写入 `plans/`。每个步骤包括上下文简报、任务列表、验证命令和退出标准 — 因此新智能体可以在不读取先前步骤的情况下执行任何步骤。
4. **审查** — 将对抗性审查委托给最强模型子智能体（例如 Opus），对照检查清单和反模式目录。在最终定稿之前修复所有关键发现。
5. **注册** — 保存计划，更新内存索引，并向用户展示步骤数和并行性摘要。

Blueprint 自动检测 git/gh 可用性。使用 git + GitHub CLI，它生成完整的分支/PR/CI 工作流计划。没有它们时，它切换到直接模式（就地编辑，无分支）。

## 示例

### 基本用法

```
/blueprint myapp "migrate database to PostgreSQL"
```

生成 `plans/myapp-migrate-database-to-postgresql.md`，包含以下步骤：
- 步骤 1：添加 PostgreSQL 驱动程序和连接配置
- 步骤 2：为每个表创建迁移脚本
- 步骤 3：更新存储库层以使用新驱动程序
- 步骤 4：添加针对 PostgreSQL 的集成测试
- 步骤 5：删除旧数据库代码和配置

### 多智能体项目

```
/blueprint chatbot "extract LLM providers into a plugin system"
```

生成一个尽可能包含并行步骤的计划（例如，在插件接口步骤完成后"实现 Anthropic 插件"和"实现 OpenAI 插件"并行运行）、模型层分配（接口设计步骤使用最强，实施使用默认），以及在每步之后验证的不变量（例如，"所有现有测试通过"、"核心中无提供商导入"）。

## 主要功能

- **冷启动执行** — 每个步骤包括独立的上下文简报。不需要先前的上下文。
- **对抗性审查网关** — 每个计划由最强模型子智能体对照检查清单进行审查，涵盖完整性、依赖正确性和反模式检测。
- **分支/PR/CI 工作流** — 内置于每个步骤中。当 git/gh 缺失时优雅地降级为直接模式。
- **并行步骤检测** — 依赖图识别没有共享文件或输出依赖的步骤。
- **计划变异协议** — 步骤可以通过正式协议和审计跟踪进行拆分、插入、跳过、重新排序或放弃。
- **零运行时风险** — 纯 Markdown 技能。整个存储库仅包含 `.md` 文件 — 无挂钩、无 shell 脚本、无可执行代码、无 `package.json`、无构建步骤。除了 Claude Code 的原生 Markdown 技能加载器之外，安装或调用时不会运行任何东西。

## 安装

此技能随 Everything Claude Code 一起提供。安装 ECC 时不需要单独安装。

### 完整 ECC 安装

如果您正在从 ECC 存储库签出工作，通过以下方式验证技能存在：

```bash
test -f skills/blueprint/SKILL.md
```

要稍后更新，请在更新之前审查 ECC 差异：

```bash
cd /path/to/everything-claude-code
git fetch origin main
git log --oneline HEAD..origin/main       # 在更新之前审查新提交
git checkout <reviewed-full-sha>          # 固定到特定的审查提交
```

### 独立 vendored 安装

如果您在完整 ECC 安装之外仅 vendored 此技能，请从 ECC 存储库中审查的文件复制到 `~/.claude/skills/blueprint/SKILL.md`。Vendored 副本没有 git 远程，因此通过从审查的 ECC 提交重新复制文件来更新它们，而不是运行 `git pull`。

## 要求

- Claude Code（用于 `/blueprint` 斜杠命令）
- Git + GitHub CLI（可选 — 启用完整的分支/PR/CI 工作流；Blueprint 检测缺失并自动切换到直接模式）
