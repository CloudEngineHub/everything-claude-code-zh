# 投资者材料

构建一致、可信且易于辩护的面向投资者的材料。

## 何时激活

- 创建或修改融资演示文稿
- 撰写投资者备忘录或一页纸
- 构建财务模型、里程碑计划或资金用途表
- 回答加速器或孵化器申请问题
- 围绕单一真实来源对齐多个融资文档

## 黄金规则

所有投资者材料必须相互一致。

在编写之前创建或确认单一真实来源：
- 吸引力指标
- 定价和收入假设
- 融资规模和工具
- 资金用途
- 团队简介和头衔
- 里程碑和时间表

如果出现冲突的数字，请在起草前解决它们。

## 核心工作流

1. 清点规范事实
2. 识别缺失的假设
3. 选择资产类型
4. 使用显式逻辑起草资产
5. 根据真实来源交叉检查每个数字

## 资产指导

### 融资演示文稿
推荐流程：
1. 公司 + 楔子
2. 问题
3. 解决方案
4. 产品 / 演示
5. 市场
6. 商业模式
7. 吸引力
8. 团队
9. 竞争对手 / 差异化
10. 请求
11. 资金用途 / 里程碑
12. 附录

如果用户想要 Web 原生演示，请将此技能与 `frontend-slides` 配对。

### 一页纸 / 备忘录
- 用一个干净的句子说明公司做什么
- 尽早展示为什么是现在
- 尽早包含吸引力和证明点
- 使请求精确
- 使声明易于验证

### 财务模型
包括：
- 显式假设
- 在有用时的熊市 / 基础 / 牛市情况
- 清晰的逐层收入逻辑
- 与里程碑关联的支出
- 决策取决于假设的敏感性分析

### 加速器申请
- 准确回答所问的问题
- 优先考虑吸引力、洞察力和团队优势
- 避免虚饰
- 保持内部指标与演示和模型一致

## 要避免的危险信号

- 无法验证的声明
- 没有假设的模糊市场规模
- 不一致的团队角色或头衔
- 收入数学不干净地求和
- 在假设脆弱的地方夸大确定性

## 质量门

在交付之前：
- 每个数字匹配当前真实来源
- 资金用途和收入层正确求和
- 假设可见，而非隐藏
- 故事清晰，没有夸大语言
- 最终资产在合伙人会议中可辩护
