# 声音特征 Schema

构建可复用的声音特征时使用此精确结构：

```text
VOICE PROFILE
=============
Author:
Goal:
Confidence:

Source Set
- source 1
- source 2
- source 3

Rhythm
- 关于句子长度、节奏和碎片化的简短说明

Compression
- 写作是密集的还是解释性的

Capitalization
- 传统的、混合的或情境性的

Parentheticals
- 如何使用以及如何不使用

Question Use
- 罕见的、频繁的、反问的、直接的或基本不存在的

Claim Style
- 主张如何构建、支持和强化

Preferred Moves
- 作者确实使用的具体手法

Banned Moves
- 作者不使用的特定模式

CTA Rules
- 如何、何时或是否以行动号召结尾

Channel Notes
- X:
- LinkedIn:
- Email:
```

指南：

- 保持特征具体且有来源依据。
- 使用简短的项目符号，而不是散文段落。
- 每个禁止手法应该可以在来源集中观察到或由用户明确要求。
- 如果来源集有冲突，指出分歧而不是平均成模糊的结果。
