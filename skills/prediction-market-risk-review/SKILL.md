# 预测市场风险审查

在预测市场工作流触及用户金融上下文、场所认证、投资组合数据、自动化或执行能力工具之前使用此技能。

## 审查关卡

### 建议边界

- 确认输出是信息性的。
- 移除买入/卖出/持有/仓位建议。
- 保持手动用户决策点的显式性。

### 场所和监管边界

- 识别场所条款、地理限制、账户限制和 API 规则。
- 在相关时标记投注、衍生品、证券或商品的模糊性以供法律审查。
- 不要绕过场所限制或速率限制。

### 数据质量

- 检查市场流动性、价差、解决规则、过时价格和来源时间戳。
- 分离公开场所数据和 Itô 门控数据。
- 不要在没有标签的情况下混合公开和私有来源。

### 安全

- 不要请求或存储私钥、种子短语或密码。
- 将 `ITO_API_KEY` 和场所 API 密钥排除在日志和文档之外。
- 默认使用只读范围。
- 在任何私有实现添加执行之前，要求熔断器、消费限制、空运行和人工批准。

### 隐私

- 最小化用户投资组合、金融和知识库数据。
- 在公共产物中编辑私有来源。
- 仅保留审查所需的字段。

## 输出契约

返回：

1. 审查范围
2. 通过/警告/失败发现
3. 被阻止的操作
4. 必需的缓解措施
5. 安全的下一步

如果请求了任何执行能力的步骤，要求单独的实现计划和明确的用户批准。
