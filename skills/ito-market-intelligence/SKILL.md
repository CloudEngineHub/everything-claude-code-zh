# Itô 市场智能

当用户需要预测市场上下文、事件发现、场所比较、篮子主题探索或 Itô API 支持的市场简报时，使用此技能。

这是一个公共预告技能。默认可以与公共来源一起使用。任何 Itô 支持的数据调用都需要通过 `ITO_API_KEY` 进行显式 API 访问。

## 防护

- 不提供投资、法律、税务或交易建议。
- 不下达、取消、路由或模拟实时订单。
- 除非用户提供，否则不要推断用户的财务状况。
- 将 Polymarket、Kalshi、Itô、X、Exa、GitHub 和 Web 数据视为来源输入，而非其本身的真实。
- 分离事实、市场隐含信号和您的解释。

## 工作流

1. 阐明市场主题、场所、地理范围和时间范围。
2. 从场所文档/API 或基于来源的研究收集公共市场数据。
3. 如果存在 `ITO_API_KEY` 且用户明确请求 Itô 数据，则仅调用只读端点并说明访问是门控的。
4. 规范化场所之间的事件、标的、流动性、费用、解决和数据延迟差异。
5. 生成决策简报：
   - 市场/事件摘要
   - 可用的场所和标的
   - 流动性和数据质量警告
   - 相关的新闻/来源上下文
   - 任何用户操作前的未解决问题

## 有用的技能链

- 使用 `deep-research` 或 `exa-search` 进行来源发现。
- 当配置 X 访问时使用 `x-api` 进行公共社交信号发现。
- 使用 `market-research` 进行市场规模、竞争对手或商业用例。
- 在任何工作流接触用户资本、投资组合数据或执行能力凭据之前使用 `prediction-market-risk-review`。

## 输出合约

默认为带有来源链接和清晰警告的紧凑简报：

```text
此市场仅供参考，并非投资或交易建议。
```

如果缺少访问权限，请说明：

```text
Itô 实时篮子/API 数据需要门控访问。在使用 Itô 支持的读取之前请求 ITO_API_KEY。
```
