# Token 预算顾问（TBA）

在 Claude 回答之前拦截响应流，为用户提供关于响应深度的选择。

## 何时使用

- 用户想要控制响应的长度或详细程度
- 用户提到 token、预算、深度或响应长度
- 用户说"简短版本"、"tldr"、"简明扼要"、"50%深度"、"详尽"等
- 任何用户想预先选择深度/详细级别的时候

**不触发条件**：用户在此会话中已设定了级别（静默保持），或答案显然只需一行。

## 工作原理

### 步骤 1 — 估算输入 Token

使用仓库的规范上下文预算启发式方法在脑中估算提示的 token 数量。

使用与 [context-budget](../context-budget/SKILL.md) 相同的校准指导：

- 散文：`词数 × 1.3`
- 代码密集或混合/代码块：`字符数 / 4`

对于混合内容，使用主要的内容类型并保持启发式估算。

### 步骤 2 — 按复杂度估算响应大小

对提示进行分类，然后应用乘数范围获取完整响应窗口：

| 复杂度   | 乘数范围 | 示例提示                                      |
|--------------|------------------|------------------------------------------------------|
| 简单       | 3x - 8x          | "X 是什么？"、是/否、单一事实                   |
| 中等       | 8x - 20x         | "X 是怎么工作的？"                                  |
| 中高  | 10x - 25x        | 带上下文的代码请求                           |
| 复杂      | 15x - 40x        | 多部分分析、比较、架构      |
| 创意     | 10x - 30x        | 故事、论文、叙事写作                  |

响应窗口 = `input_tokens × mult_min` 到 `input_tokens × mult_max`（但不超过你模型配置的输出 token 限制）。

### 步骤 3 — 呈现深度选项

在回答之前呈现此区块，使用实际估算的数字：

```
正在分析你的提示...

输入：~[N] token  |  类型：[类型]  |  复杂度：[级别]  |  语言：[语言]

选择你的深度级别：

[1] 精要     (25%)  ->  ~[token]   仅直接回答，无前言
[2] 适中     (50%)  ->  ~[token]   回答 + 背景 + 1 个示例
[3] 详细     (75%)  ->  ~[token]   完整回答带替代方案
[4] 详尽    (100%)  ->  ~[token]   一切，无限制

选哪个级别？(1-4 或说 "25% 深度"、"50% 深度"、"75% 深度"、"100% 深度")

精度：启发式估算约 85-90% 准确率 (±15%)。
```

级别 token 估算（在响应窗口内）：
- 25%  → `min + (max - min) × 0.25`
- 50%  → `min + (max - min) × 0.50`
- 75%  → `min + (max - min) × 0.75`
- 100% → `max`

### 步骤 4 — 按所选级别响应

| 级别            | 目标长度       | 包含                                             | 省略                                              |
|------------------|---------------------|-----------------------------------------------------|---------------------------------------------------|
| 25% 精要    | 最多 2-4 句话   | 直接回答、关键结论                       | 背景、示例、细微差别、替代方案           |
| 50% 适中     | 1-3 段落      | 回答 + 必要背景 + 1 个示例              | 深度分析、边界情况、参考文献             |
| 75% 详细     | 结构化响应 | 多个示例、优缺点、替代方案          | 极端边界情况、详尽参考文献         |
| 100% 详尽  | 无限制      | 一切 — 完整分析、所有代码、所有视角 | 无                                        |

## 快捷方式 — 跳过提问

如果用户已经表明了级别，直接按该级别响应而不询问：

| 用户说的                                      | 级别 |
|----------------------------------------------------|-------|
| "1" / "25% 深度" / "简短版本" / "简明回答" / "tldr"  | 25%   |
| "2" / "50% 深度" / "适中深度" / "平衡回答"        | 50%   |
| "3" / "75% 深度" / "详细回答" / "全面回答"       | 75%   |
| "4" / "100% 深度" / "详尽回答" / "完整深挖"     | 100%  |

如果用户在此会话早些时候设定了级别，在后续响应中**静默保持**，除非他们更改它。

## 精度说明

此技能使用启发式估算 — 没有真正的分词器。准确率约 85-90%，方差 ±15%。始终显示免责声明。

## 示例

### 触发条件

- "先给我简短版本。"
- "你的回答会用多少 token？"
- "以 50% 深度回答。"
- "我想要详尽回答，不是摘要。"
- "先给简短版再给详细版。"

### 不触发

- "JWT token 是什么？"
- "结账流程使用支付 token。"
- "这正常吗？"
- "完成重构。"
- 用户已为此会话选择深度后的后续问题
