# 渲染策略

Angular 支持多种渲染策略以优化 SEO、性能和交互性。

## 1. 客户端渲染（CSR）

**默认策略。** 内容完全在浏览器中渲染。

- **用例**：交互式仪表板、内部工具。
- **优点**：配置最简单，服务器成本低。
- **缺点**：SEO 较差，初始内容可见性较慢（必须等待 JS）。

## 2. 静态站点生成（SSG / 预渲染）

内容在**构建时**预渲染为静态 HTML 文件。

- **用例**：营销页面、博客、文档。
- **优点**：最快的初始加载、出色的 SEO、CDN 友好。
- **缺点**：内容更新需要重新构建，不适用于用户特定数据。

## 3. 服务端渲染（SSR）

内容在**初始请求**时在服务器上渲染。后续导航以客户端方式进行（SPA 风格）。

- **用例**：电商产品页面、新闻网站、个性化动态内容。
- **优点**：出色的 SEO、快速的初始内容可见性。
- **缺点**：需要服务器（Node.js）、更高的服务器成本/延迟。

## 水合（Hydration）

水合是在浏览器中使服务端渲染的 HTML 变为可交互的过程。

- **完全水合**：整个应用同时变为可交互。
- **增量水合**：（高级）使用 `@defer` 块按需使部分变为可交互。
- **事件回放**：捕获并回放在水合完成之前发生的用户事件。

## 决策矩阵

| 需求                     | 策略             |
| :----------------------- | :--------------- |
| **SEO + 静态内容**       | SSG              |
| **SEO + 动态内容**       | SSR              |
| **无 SEO + 高交互性**    | CSR              |
| **混合**                 | 混合（基于路由）  |
