# Itô Data Atlas 代理

使用此技能设计一个代理，监视数据源、构建候选预测市场篮子、起草参数变更，并将结果交给人工审查。

此技能描述架构和工作流。它不运行实时交易。

## 防护

- 将所有执行保持在明确的人工批准之后。
- 仅在单独的私有实现明确添加执行控制时才需要 `ITO_API_KEY` 进行只读 Itô 数据访问。
- 除非目标存储库已有存储合约且用户要求，否则不要持久化私有用户数据。
- 不要在公共文档中暴露私有策略逻辑、场所凭据或本地路径。

## 架构模式

使用四条通道：

1. 研究收集器：公共 Web、X、GitHub、场所文档、API 元数据，以及在存在门控访问时的 Itô 只读端点。
2. 篮子起草人：将来源转换为候选标的、权重、规则和问题。
3. 风险审查者：检查数据新鲜度、场所限制、解决歧义、合规说明和提示注入暴露。
4. 人工编辑器：打开聊天或 UI 状态，用户可以在其中批准、拒绝、调整或请求更多研究。

## 工作流

1. 定义用户目标和排除的操作。
2. 列出数据源和访问要求。
3. 起草具有每个标的来源的篮子规范。
4. 生成可编辑参数而非可执行订单。
5. 存储审计跟踪：输入、模型输出、来源和人工决策。

## 有用的技能链

- `deep-research` 用于来源收集。
- `x-api` 用于当前社交/事件信号。
- `ito-market-intelligence` 用于场所和标的上下文。
- `ito-basket-compare` 用于用户知识库匹配。
- `prediction-market-risk-review` 在任何具有执行能力的集成之前。

## 输出合约

返回实现就绪的工作流规范，包含：

- 数据源
- 访问门控
- 代理角色
- 人工批准点
- 存储/审计边界
- 非目标
