# 内容引擎

构建平台原生的内容，而不将作者的真实声音压扁成平台的平庸模板。

## 何时激活

- 撰写 X 帖子或推文串
- 起草 LinkedIn 帖子或发布更新
- 撰写短视频脚本或 YouTube 解释性视频
- 将文章、播客、演示、文档或内部笔记复用为公开内容
- 围绕产品、洞察或叙事构建发布序列或持续内容系统

## 不可妥协的原则

1. 从原始素材出发，而非通用帖子模板。
2. 为平台适配格式，而非改变人格。
3. 每条帖子应该承载一个真正的观点。
4. 具体性胜过形容词堆砌。
5. 不使用互动诱饵，除非用户明确要求。

## 素材优先工作流

起草前，先确定素材集：
- 已发布的文章
- 笔记或内部备忘录
- 产品演示
- 文档或变更日志
- 文字稿
- 截图
- 同一作者之前的帖子

如果用户需要特定的声音，在写作之前从真实示例构建声音画像。
当声音一致性对多个输出重要时，使用 `brand-voice` 作为标准工作流。

## 声音处理

`brand-voice` 是标准的声音层。

在以下情况下首先运行它：

- 有多个下游输出
- 用户明确关心写作风格
- 内容是发布、外联或声誉敏感的

在此复用生成的 `VOICE PROFILE`，而非重建第二个声音模型。
如果用户特别想要 Affaan / ECC 的声音，仍然将 `brand-voice` 视为真相来源，并使用最佳的实时或素材派生材料。

## 硬性禁止

删除并重写以下内容：
- "In today's rapidly evolving landscape"
- "game-changer"、"revolutionary"、"cutting-edge"
- "here's why this matters" 除非后面紧跟具体内容
- 以 LinkedIn 风格的问题结尾仅为收割回复
- LinkedIn 上的虚假随意感
- 原始素材中不存在的虚假互动填充

## 平台适配规则

### X

- 以最强的观点、作品或张力开场
- 如果原始声音是简洁的，保持压缩感
- 如果写推文串，每条帖子必须推进论点
- 不要填充受众不需要的背景信息

### LinkedIn

- 仅扩展到让非直接领域的人能跟上即可
- 除非原始素材确实是反思性的，否则不要变成虚假的课程帖
- 不要企业励志节奏
- 不要捧场堆叠，不要"旅程"填充

### 短视频

- 围绕视觉序列和证据点撰写脚本
- 前几秒应展示结果、问题或亮点
- 不要写出在纸上比在屏幕上听起来更好的旁白

### YouTube

- 早期展示结果或张力
- 按论点或递进组织，而非填充性章节
- 仅在有助于清晰度时使用章节划分

### 新闻通讯

- 以要点、冲突或作品开场
- 不要花第一段来暖场
- 每个部分都需要添加新内容

## 复用流程

1. 选择锚定素材。
2. 提取 3 到 7 个原子观点或场景。
3. 按锐度、新颖性和证据排名。
4. 每个输出分配一个强观点。
5. 为每个平台适配结构。
6. 剥离平台形态的填充内容。
7. 运行质量关卡。

## 交付物

当被要求制作营销活动时，返回：
- 如果声音匹配重要，提供简短的声音画像
- 核心角度
- 平台原生的草稿
- 仅在有助于执行时提供发布顺序
- 发布前必须填补的空缺

## 质量关卡

交付前：
- 每个草稿听起来像目标作者，而非平台刻板印象
- 每个草稿包含真实的观点、证据点或具体观察
- 没有剩余的通用炒作语言
- 没有剩余的虚假互动诱饵
- 跨平台没有重复的文案（除非被要求）
- 任何 CTA 都是应得的且经用户批准的

## 相关技能

- `brand-voice` 用于从素材派生的声音画像
- `crosspost` 用于平台特定的分发
- `x-api` 用于获取最近帖子和发布已批准的 X 输出
