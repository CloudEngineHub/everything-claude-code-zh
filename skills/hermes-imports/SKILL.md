# Hermes 导入

在将重复的 Hermes 工作流转换为可以在 ECC 中安全发布的内容时使用此技能。

Hermes 是操作员 shell。ECC 是可重用工作流层。导入应将稳定模式从 Hermes 移至 ECC，而不移动私有状态。

## 何时使用

- Hermes 工作流已重复足够多次变得可重用。
- 本地操作员提示应成为公共 ECC 技能。
- 发布、内容、研究或工程工作流需要经过清理的交接文档。
- 工作流提及本地路径、凭据、个人数据集或私有账户名称，这些必须在发布前删除。

## 导入规则

- 将本地路径转换为仓库相对路径或占位符。
- 使用角色标签（如 `operator`、`default profile` 或 `workspace owner`）替换实时账户名称。
- 仅按提供商名称描述凭据要求。
- 保持示例狭窄且可操作。
- 不要发布原始工作区导出、令牌、OAuth 文件、健康数据、CRM 数据或财务数据。
- 如果工作流需要私有状态才有意义，请保持本地。

## 清理检查清单

在提交导入的工作流之前，扫描以下内容：

- 绝对路径，如 `/Users/...`
- `~/.hermes` 路径，除非文档明确说明本地设置
- API 密钥、令牌、cookies、OAuth 文件或 bearer 字符串
- 电话号码、私人电子邮件地址和个人联系人图
- 尚未公开的客户名称、家庭名称或账户名称
- 收入、健康或 CRM 详细信息
- 包含来自私有系统的工具输出的原始日志

## 转换模式

1. 识别可重复的操作员循环。
2. 剥离私有输入和输出。
3. 将本地路径重写为仓库相对示例。
4. 将一次性指令转换为"何时使用"部分和简短流程。
5. 添加具体的输出要求。
6. 在打开 PR 之前运行密钥和本地路径扫描。

## 示例：发布交接

本地 Hermes 提示：

```text
阅读我的本地工作区文件并定稿发布文案。
```

ECC 安全版本：

```text
使用 docs/releases/<version>/ 下的公共发布包。
返回一个 X 帖子、一个 LinkedIn 帖子、一个录制检查清单和缺失资产列表。
```

## 示例：安静时段操作员任务

本地 Hermes 任务：

```text
过夜运行我的私人收件箱、财务和内容检查。
```

ECC 安全版本：

```text
描述调度器策略、安静时段窗口、升级规则和检查类别。不要包括私有数据源或凭据。
```

## 输出合约

返回：

- 候选 ECC 技能名称
- 经过清理的工作流摘要
- 所需的公共输入
- 已删除的私有输入
- 剩余风险
- 应创建或更新的文件
