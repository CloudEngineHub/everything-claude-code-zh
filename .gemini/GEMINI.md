# Gemini CLI 的 ECC

此文件为 Gemini CLI 提供基准 ECC 工作流、审查标准和安全检查，适用于安装 Gemini 目标的仓库。

## 概述

Everything Claude Code (ECC) 是一个跨 harness 编码系统，具有 36 个专业代理、142 个技能和 68 个命令。

Gemini 支持目前专注于通过 `.gemini/GEMINI.md` 提供强大的项目本地指令层，加上安装程序提供的共享 MCP 目录和包管理器设置资产。

## 核心工作流

1. 在编辑大型功能之前规划。
2. 对于错误修复和新功能，优先测试优先的更改。
3. 在发布之前审查安全性。
4. 保持更改自包含、可读且易于恢复。

## 编码标准

- 优先使用不可变更新，而不是就地变异。
- 保持函数小且文件专注。
- 在边界处验证用户输入。
- 永远不要硬编码机密。
- 使用清晰的错误消息大声失败，而不是默默地吞下问题。

## 安全检查清单

在任何提交之前：

- 无硬编码 API 密钥、密码或令牌
- 验证所有外部输入
- 数据库写入使用参数化查询
- 在适用情况下清理 HTML 输出
- 敏感路径检查 authz/authn
- 错误消息已删除敏感内部信息

## 交付标准

- 使用传统提交：`feat`、`fix`、`refactor`、`docs`、`test`、`chore`、`perf`、`ci`
- 在发布前对接触区域运行目标验证
- 优先使用包含的本地实现，而不是添加新的第三方运行时依赖

## 要重用的 ECC 区域

- `AGENTS.md` 用于仓库范围的操作规则
- `skills/` 用于深度工作流指导
- `commands/` 用于值得适应到提示/宏的斜杠命令模式
- `mcp-configs/` 用于共享连接器基线
