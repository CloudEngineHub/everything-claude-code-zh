一个结构化的功能开发工作流，强调在编写新代码之前理解现有代码。

## 阶段

### 1. 发现

- 仔细阅读功能需求
- 确定需求、约束和验收标准
- 如果需求模糊，提出澄清问题

### 2. 代码库探索

- 使用 `code-explorer` 分析相关的现有代码
- 追踪执行路径和架构层次
- 理解集成点和约定

### 3. 澄清问题

- 呈现探索中的发现
- 提出有针对性的设计和边缘情况问题
- 等待用户响应后再继续

### 4. 架构设计

- 使用 `code-architect` 设计功能
- 提供实现蓝图
- 等待批准后再实现

### 5. 实现

- 按批准的设计实现功能
- 在适当处优先使用 TDD
- 保持提交小而聚焦

### 6. 质量审查

- 使用 `code-reviewer` 审查实现
- 解决关键和重要问题
- 验证测试覆盖

### 7. 总结

- 总结构建的内容
- 列出后续事项或限制
- 提供测试说明
