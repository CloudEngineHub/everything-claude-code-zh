# 重构清理 (Refactor Clean)

安全地识别和移除死代码，每一步都通过测试验证。

## 步骤 1：检测死代码

根据项目类型运行分析工具：

| 工具 | 检测内容 | 命令 |
|------|--------------|---------|
| knip | 未使用的导出、文件、依赖项 | `npx knip` |
| depcheck | 未使用的 npm 依赖项 | `npx depcheck` |
| ts-prune | 未使用的 TypeScript 导出 | `npx ts-prune` |
| vulture | 未使用的 Python 代码 | `vulture src/` |
| deadcode | 未使用的 Go 代码 | `deadcode ./...` |
| cargo-udeps | 未使用的 Rust 依赖项 | `cargo +nightly udeps` |

如果没有工具可用，使用 Grep 查找零导入的导出：
```
# 查找导出，然后检查它们是否在任何地方被导入
```

## 步骤 2：分类发现

将发现按安全级别分类：

| 级别 | 示例 | 操作 |
|------|----------|--------|
| **SAFE** | 未使用的工具函数、测试辅助函数、内部函数 | 可放心删除 |
| **CAUTION** | 组件、API 路由、中间件 | 验证没有动态导入或外部消费者 |
| **DANGER** | 配置文件、入口点、类型定义 | 修改前需深入调查 |

## 步骤 3：安全删除循环

对于每个 SAFE 项：

1. **运行完整测试套件** — 建立基线（全部通过）
2. **删除死代码** — 使用 Edit 工具进行精确移除
3. **重新运行测试套件** — 验证没有破坏
4. **如果测试失败** — 立即用 `git checkout -- <file>` 回滚并跳过此项
5. **如果测试通过** — 继续下一项

## 步骤 4：处理 CAUTION 项

在删除 CAUTION 项之前：
- 搜索动态导入：`import()`、`require()`、`__import__`
- 搜索字符串引用：路由名称、配置中的组件名称
- 检查是否从公共包 API 导出
- 验证没有外部消费者（如果已发布则检查依赖者）

## 步骤 5：合并重复项

移除死代码后，查找：
- 近重复函数（>80% 相似）— 合并为一个
- 冗余类型定义 — 合并
- 不增加价值的包装函数 — 内联
- 无目的的重新导出 — 移除间接层

## 步骤 6：摘要

报告结果：

```
Dead Code Cleanup
──────────────────────────────
Deleted:   12 unused functions
           3 unused files
           5 unused dependencies
Skipped:   2 items (tests failed)
Saved:     ~450 lines removed
──────────────────────────────
All tests passing PASS:
```

## 规则

- **绝不未运行测试就删除代码**
- **一次删除一项** — 原子化更改使回滚容易
- **不确定就跳过** — 保留死代码比破坏生产环境好
- **清理时不要重构** — 分离关注点（先清理，后重构）
